203 Non-authoritative Information
=================================

The request was successful but the enclosed payload has been modified from that of the origin server's [200 OK](https://httpstatuses.com/200) response by a transforming proxy[1](#ref-1).

This status code allows the proxy to notify recipients when a transformation has been applied, since that knowledge might impact later decisions regarding the content. For example, future cache validation requests for the content might only be applicable along the same request path (through the same proxies).

The 203 response is similar to the Warning code of 214 Transformation Applied[2](#ref-2), which has the advantage of being applicable to responses with any status code.

A 203 response is cacheable by default; i.e., unless otherwise indicated by the method definition or explicit cache controls[3](#ref-3).

*   1 Transformations [RFC7230 Section 5.7.2](https://tools.ietf.org/html/rfc7230#section-5.7.2)
*   2 Warning [RFC7234 Section 5.5](https://tools.ietf.org/html/rfc7234#section-5.5)
*   3 Calculating Heuristic Freshness [RFC7234 Section 4.2.2](https://tools.ietf.org/html/rfc7234#section-4.2.2)
*   Source: [RFC7231 Section 6.3.4](https://tools.ietf.org/html/rfc7231#section-6.3.4)
